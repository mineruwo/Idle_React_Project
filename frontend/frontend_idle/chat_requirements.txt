# Frontend (React) - 실시간 채팅 기능 요구사항

## 1. UI/UX 디자인 및 컴포넌트
- **채팅방 목록:**
    - 유저-상담원 채팅방 목록
    - 유저-유저 채팅방 목록
    - 채팅방 이름, 마지막 메시지, 읽지 않은 메시지 수 표시
- **채팅방 화면:**
    - 메시지 표시 영역 (스크롤 가능)
    - 메시지 입력창 (텍스트 입력, 이모티콘, 파일 첨부 등)
    - 메시지 전송 버튼
    - 상대방 접속 상태 표시
- **상담 요청/시작 UI (유저-상담원 채팅):**
    - 상담 요청 버튼
    - 상담원 연결 대기 화면
- **채팅방 생성 UI (유저-유저 채팅):**
    - 채팅방 이름 입력
    - 참여자 초대 기능

## 2. 실시간 통신 클라이언트
- WebSocket 클라이언트 구현 (Socket.IO 클라이언트 라이브러리 등)
- 백엔드 WebSocket 서버와 연결 및 메시지 송수신
- 연결 상태 관리 (연결 중, 연결 끊김 등)

## 3. 상태 관리
- Redux 또는 Context API를 활용한 전역 상태 관리
- 채팅방 목록, 현재 활성화된 채팅방, 메시지 목록, 사용자 입력 상태 등

## 4. API 연동
- 백엔드 API 호출을 통한 채팅방 생성, 조회, 참여, 나가기 등
- 메시지 히스토리 로드

## 5. 사용자 경험 개선
- 메시지 전송 시 스크롤 자동 이동
- 메시지 읽음 처리
- 알림 기능 (새 메시지 도착 시)
- 로딩 스피너/인디케이터

## 6. 라우팅
- 채팅방별 고유 URL (예: /chat/:roomId)
- 상담 요청 페이지, 채팅방 목록 페이지 등
